# WANDO MCP — ставь задачи в WANDO прямо из Claude

Маленький [MCP](https://modelcontextprotocol.io)-сервер. Даёт Claude инструмент
`cockpit_propose_task` — задача из чата улетает в очередь **«На приёмке»** на
[cock-pit.com](https://cock-pit.com), где ты её подтверждаешь, правишь, кидаешь в
бэклог или отклоняешь. Напрямую ничего не создаётся — человек всегда решает.

Работает в **Claude Code (VS Code)** и в **Claude Desktop** — один и тот же сервер.

Двусторонний: Claude умеет не только **ставить** задачи (`cockpit_propose_task`), но и **читать статус** проекта (`cockpit_status`) — спроси «что по проекту / что горит в WANDO» прямо в чате.

---

## 1. Получи персональный ключ

Открой **cock-pit.com → меню ⋯ → «Подключить Claude»** → «Сгенерировать ключ».
Скопируй ключ `cpk_…` (показывается один раз). Ключ привязан к тебе — у Димы свой.

## 2. Подключи

### Claude Code (VS Code) — Роч
```bash
claude mcp add cockpit \
  --env COCKPIT_TOKEN=cpk_ВАШ_КЛЮЧ \
  --env COCKPIT_SOURCE=claude-code \
  -- npx -y cockpit-tasks-mcp
```
(или укажи путь: `-- node /путь/к/cockpit/mcp/cockpit-mcp/index.js`)

### Claude Desktop — Дима
Settings → Developer → Edit Config, добавь в `mcpServers`:
```json
{
  "mcpServers": {
    "cockpit": {
      "command": "npx",
      "args": ["-y", "cockpit-tasks-mcp"],
      "env": {
        "COCKPIT_TOKEN": "cpk_ВАШ_КЛЮЧ",
        "COCKPIT_SOURCE": "claude-desktop"
      }
    }
  }
}
```
Перезапусти Claude Desktop.

## 3. Пользуйся — быстрые команды

Начинай с триггера **«WANDO:»** или **«в WANDO:»**, дальше обычным языком:

| Скажи Claude | Что получится |
|---|---|
| `WANDO: созвон с Димой завтра 2ч срочно` | задача на завтра, 2ч, ⚑срочно |
| `в WANDO: вёрстка лендинга с 22 по 28 июня 12ч @дима` | срок-диапазон, оценка, исполнитель |
| `WANDO, проект AppHub: настроить аналитику на 3 дня важно` | в конкретный проект |
| `Закинь в WANDO три задачи: A, B, C` | три заявки разом |
| `Сделай из этого задачу в WANDO` | Claude сам сформулирует из обсуждения |

Распознаётся автоматически: **кому** `@имя` / «для Димы» · **срок** сегодня/завтра/послезавтра /
«с 22 по 28 июня» / «на 3 дня» · **оценка** `2ч` `12ч` · **важность** `!срочно` `важно` ·
**проект** «в проект &lt;имя&gt;».

Заявка появляется на сайте во вкладке **Приёмка** (и в 🔔) — там подтверждаешь.

**Спросить статус** (только чтение, ничего не меняет):

| Скажи Claude | Что получится |
|---|---|
| `Что горит в WANDO?` | по каждому проекту: готово/всего, что просрочено, ближайшее |
| `Какой статус по проектам?` | компактная сводка команды |
| `Над чем мне работать дальше?` | Claude смотрит очередь и предлагает шаг |

> Для `cockpit_status` нужно один раз применить `supabase/status.sql` в Supabase (как и `intake.sql`).

## Переменные окружения
| Переменная | Обязательна | По умолчанию | Зачем |
|---|---|---|---|
| `COCKPIT_TOKEN` | да | — | твой ключ `cpk_…` |
| `COCKPIT_URL` | нет | общий проект WANDO | URL проекта Supabase (для своего инстанса) |
| `COCKPIT_ANON` | нет | встроен | публичный anon-ключ Supabase (для своего инстанса) |
| `COCKPIT_SOURCE` | нет | `claude` | метка устройства в заявке |

## Безопасность
- Ключ хранится у тебя локально (в конфиге Claude), в базе — только его SHA-256-хэш.
- Ключ можно отозвать в любой момент на сайте → старое подключение перестаёт работать.
- MCP умеет только **предлагать** задачи. Изменять/удалять данные он не может.
