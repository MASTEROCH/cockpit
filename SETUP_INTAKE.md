# Активация приёмника Claude → WANDO

Вся серверная часть — один SQL-файл. Никаких функций деплоить не надо.

## 1. База (один раз)
Supabase → проект WANDO → **SQL Editor** → вставь и выполни весь файл
[`supabase/intake.sql`](supabase/intake.sql).

Создаёт:
- таблицы `intake` (очередь заявок) и `intake_tokens` (персональные ключи) + RLS + realtime;
- функцию `cockpit_intake(...)` — защищённый приём заявок от Claude (проверяет ключ сама,
  поэтому вызывается публичным anon-ключом, секреты не нужны);
- функцию `cockpit_projects(...)` — список проектов для Claude.

## 2. Подключить Claude (каждому — себе)
На **cock-pit.com** войди → меню **⋯ → 🔌 Подключить Claude** → «Сгенерировать ключ»
→ скопируй конфиг (вкладка Claude Code или Claude Desktop) → вставь у себя.

- **Роч (VS Code):** `claude mcp add cockpit …` (команда уже с твоим ключом в модалке)
- **Дима (Desktop):** вставить JSON в Settings → Developer → Edit Config, перезапустить

Детали MCP-сервера: [`mcp/cockpit-mcp/README.md`](mcp/cockpit-mcp/README.md).

## 3. Как пользоваться
Скажи своему Claude:
> «Поставь в WANDO задачу: созвон с Димой завтра 2ч, срочно»

Заявка прилетает на сайт во вкладку **Приёмка** (и в 🔔 колокольчик). Жмёшь
**✓ Добавить / ✎ Править / 📥 В бэклог / ✕ Отклонить**. Текст распознаётся умным
парсером: `@имя`, `завтра` / `с 22 по 28 июня`, `2ч` / `12ч`, `!срочно` / `важно`.

## Как это устроено (безопасность)
```
Claude → MCP cockpit-mcp ──anon-ключ + твой cpk_-токен──► RPC cockpit_intake()
                                                            (SECURITY DEFINER: проверяет токен,
                                                             пишет в intake от имени владельца)
                                                                  │ realtime
                                                            cock-pit.com → Приёмка → ты решаешь
```
- Ключи `cpk_…` персональные, в базе только их SHA-256-хэш, отзываются на сайте.
- Писать в очередь может только функция (у клиента нет RLS-insert).
- anon-ключ публичный (он и так в сайте) — сам по себе ничего не даёт без валидного `cpk_`-токена.

## Для продажи другим фаундерам
Схема уже мультитенант-ready: у токенов и заявок есть `workspace` (сейчас у всех `default`).
Выдавай каждому клиенту свой `workspace` — приёмки изолируются автоматически (RLS уже по workspace).
MCP публикуется в npm как `cockpit-tasks-mcp`: клиент ставит одной строкой и вставляет свой ключ.
