# WANDO · Telegram-бот — запуск за 5 минут

Задачи в WANDO прямо из Telegram (с телефона, на ходу). Ничего не ставится:
бот живёт в твоём же Supabase как edge-функция. Каждый пишет со своего
аккаунта со своим ключом; всё падает в **Приёмку** — подтверждает команда.

```
Дима/Денис в TG: «созвон завтра 2ч срочно»
   → tg-bot (edge-функция) → таблица intake → Приёмка на cock-pit.com → ✓/✕
«/status» → сводка проектов (команда видит всё, гость — только выданное)
```

## Шаг 0 · SQL (один раз, если ещё не прогнал team.sql)
Supabase → SQL Editor → выполни [`supabase/team.sql`](supabase/team.sql)
(⚠️ сначала замени `DENIS_EMAIL_HERE` на почту Дениса). Это создаст
`team`, `project_access`, `tg_links` и обновит права.

## Шаг 1 · Создай бота (1 мин)
1. В Telegram открой **@BotFather** → `/newbot`
2. Имя: `WANDO` · юзернейм: например `wando_tasks_bot`
3. Скопируй **токен** вида `1234567890:AAE...` — никуда не пересылай, он нужен только в Supabase

## Шаг 2 · Секреты (1 мин)
Supabase Dashboard → **Edge Functions → Secrets** → добавь:
| Name | Value |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен из шага 1 |
| `TG_WEBHOOK_SECRET` | любая своя случайная строка, напр. `wando_hook_8Fk2mQ91` |

## Шаг 3 · Деплой функции (2 мин)
Dashboard → **Edge Functions → Deploy new function** → имя: `tg-bot`
→ вставь содержимое [`supabase/functions/tg-bot/index.ts`](supabase/functions/tg-bot/index.ts)
→ **ВЫКЛЮЧИ «Verify JWT»** (Telegram не шлёт Authorization; вместо него
функция проверяет `TG_WEBHOOK_SECRET`) → Deploy.

## Шаг 4 · Вебхук (30 сек)
В терминале (подставь СВОЙ токен и СВОЙ секрет из шага 2):
```bash
curl -s "https://api.telegram.org/bot<ТОКЕН>/setWebhook" \
  -d "url=https://tonmsmxzmycimybzywqp.supabase.co/functions/v1/tg-bot" \
  -d "secret_token=<TG_WEBHOOK_SECRET>"
```
Ответ должен быть `{"ok":true,...,"description":"Webhook was set"}`.

## Шаг 5 · Каждый привязывает себя (30 сек на человека)
1. cock-pit.com → меню **⋯ → 🔌 Подключить Claude** → «Сгенерировать ключ» → копируешь `cpk_…`
   (ключ универсальный: работает и для Claude, и для бота)
2. В Telegram боту: `/key cpk_…` — бот привяжет и **сотрёт сообщение с ключом**
3. Готово. Пиши задачи текстом, `/status` — сводка

## Что умеет
| Пишешь боту | Что происходит |
|---|---|
| `созвон с Димой завтра 2ч срочно` | заявка в Приёмку (умный разбор на сайте) |
| `/status` · `что горит` | сводка: готово/просрочено/ближайшее по проектам |
| `/help` | справка с примерами |

## Безопасность
- Бот **ничего не создаёт напрямую** — только очередь Приёмки, решает человек
- Ключ отзывается на сайте → привязка мгновенно умирает
- Гость (уровень «Саша») в `/status` видит только выданные ему проекты
- Вебхук защищён секретом; токен бота живёт только в Supabase Secrets
